# Account Federation

## Purpose

Account federation describes when a local account, identity, or profile may be recognized across systems, devices, services, or governance domains.

## Boundary

Federation must be explicitly established. A local account profile is not external login authority, cross-device authority, public identity authority, or production federation.

## Required Inputs

A federation claim should identify:

- local account profile
- external system or domain
- actor requesting federation
- accepted identity evidence
- policy reference
- delegation reference when applicable
- validity window
- revocation path
- receipt output

## Non-Claims

Federation does not follow automatically from:

- local prototype readiness
- shell readiness
- account session readiness
- guardian documentation
- public discovery readiness

## Standing Rule

Federation standing must be evaluated against current policy, delegation, evidence, and context before use.
